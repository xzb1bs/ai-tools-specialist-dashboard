# AI Tools Specialist — Тестовое задание

## Что было сделано

- Загружены 50 тестовых заказов в RetailCRM через API (`/api/v5/orders/upload`)
- Настроена синхронизация заказов из RetailCRM в Supabase
- Создан мини-дашборд на Next.js с визуализацией данных (статистика + график)
- Дашборд успешно задеплоен на Vercel
- Реализован механизм уведомлений в Telegram при заказах больше 50 000 ₸ (через polling)

### Ссылки

- **Дашборд**: https://dashboard-beryl-two-84.vercel.app


## Технологический стек

- RetailCRM API v5
- Supabase (PostgreSQL + REST API)
- Next.js 16 + TypeScript + Tailwind
- Recharts (для графиков)
- Vercel (деплой)
- Telegram Bot API

## Что получилось

- Успешная загрузка mock-данных в RetailCRM
- Полная синхронизация заказов RetailCRM → Supabase
- Красивый и функциональный дашборд с ключевыми метриками и визуализацией
- Автоматическая проверка новых заказов через API route (`/api/check-orders`)
- Подготовлена инфраструктура для уведомлений в Telegram

## Что не получилось

К сожалению, **не удалось получить уведомление в Telegram**.

**Причина:**  
В использованном демо-аккаунте RetailCRM отсутствует возможность настройки исходящих вебхуков (раздел "Исходящие вебхуки" недоступен). Из-за этого пришлось реализовать проверку заказов через polling (`/api/check-orders`). 

Polling работает технически (endpoint отвечает), но уведомления в Telegram не приходили во время тестирования (возможно, из-за ограничений демо-аккаунта или частоты проверок).
